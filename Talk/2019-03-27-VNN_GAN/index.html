A Global Correctness Criterion using Generative Models

Image classifier

$$f : \textit{Images} \to \left\{\textit{Panda},\textit{Monkey}\right\}$$

What's wrong?

local

 check for each image of the test set


perturbations

 perturbations (often) defined using norms

Criterion

Generative model

$$g : \textit{Random Seeds} \to \textit{Panda Images}$$ thispersondoesnotexist.com
$f$ image classifier, $g$ generative model for panda images $$g \circ f : \textit{Random Seeds} \to \left\{\textit{Panda},\textit{Monkey}\right\}$$
Definition: Correctness of $f$ against $g$ wrt panda class: $$\mathbb{P}_{x \sim D}(g \circ f(x) = \text{ Panda})$$

How to check?

Given a NN $h$ with binary output, compute or approximate: $$\mathbb{P}_{x \sim D}(h(x) = 1)$$
Solution 1: Verification approach yields safe under-approximation $$\forall x, h(x) = 1$$
Solution 2: Statistical approach yields good and cheap approximations

How good is the criterion?

Global: from thousands to billions of images in the test set

Makes a key assumption: generative models are good!
Idea: create standard. Good for benchmarks and certification

Perturbations: it seems to yield a (rather) convincing way of perturbing images video

Experiments

  • small generative models yield irrelevant criteria
  • most image classifiers I could train or find are below 50% (statistical approach), but (some of) the robust ones are better
  • the correctness criterion seems to be independent of the generative model!
